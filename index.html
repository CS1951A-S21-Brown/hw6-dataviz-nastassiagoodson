For the 3rd question, since that question does not ask for a specific number of top countries, I simply selected a number of countries that seemed reasonable and would not make the information difficult to understand or read. 2. D3 was helpful in that it is relatively easy to make things interactive, so implementing the tooltip and buttons was relatively simple. In addition, d3 allows a very similar interface for different types of graphs, so it is relatively simple to use one type of graph as a basis for another one. Finally, because d3 is more frontend-oriented it can be more easily integrated with css styles in order to overall make a moer aesthetically pleasing dashboard.

3. One reason I might not want to use d3 is that it doesn't have very good documentation, so some things, especially the map, were really difficult to create correctly. Next, formatting in d3 can also be difficult, and I struggled with trying to get the labels to show up in the correct spot. Finally, because d3 is a javascript library, it may not be as efficient as using some more backend solutions, such as matplotlib in python.

4. Overall, most users should find this dashboard to be accessible. However, it's possible that especially in terms of the map I created, users who have some difficulty seeing may not be able to get the same information out of it as other users. Specifically, the map requires someone to mouseover the countries that are colored in, and then view the popover that shows up. If a user is unable to distinguish color or is unable to view the map at all, they may have trouble finding the relevant countries or may not be able to at all. In order to make this map specifically more accessible, I might add a short description in list form of the top 10 countries so that even users who have difficulty seeing or distinguishing colors will be able to get the full information out of it (e.g. alt-text of some kind), which is one technique mentioned in "A Comprehensive Guide to Accessible Data Visualization." In addition, users who have difficult seeing may not be able to use the button to select the performance metric, so they would not be able to compare the differences between the other two metrics as other users would. The only way I can think of to mitigate this is to simply add another 2 graphs so that there is one graph of each kind for every performance metric. In this way, all users may be able to get more information from the visualizations. One change I made after reading "A Comprehensive guide to Accessible Data Visualization" was to remove the mouseover I had originally added to my bar graph that would show the specific game count for that year. Instead, I simply put the count above the corresponding bar so that all users would be able to understand the graph fully.
